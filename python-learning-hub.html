Beginner Variables, data types & operators

Variables are like labelled boxes that store information. Python figures out the type automatically — no need to declare it.

name = "Alice"        # string (text)
age = 28              # integer (whole number)
salary = 7500.50      # float (decimal)
is_employed = True    # boolean (True/False)

# f-strings — used constantly in real code
print(f"{name} is {age} years old, earning RM{salary}")

Why it matters: Every program starts here. Analysts store data in variables. Developers use them to hold API responses, user inputs, config values.

Beginner Lists, dicts & loops

Lists hold ordered items. Dicts hold key-value pairs. Loops let you repeat actions over data — you'll do this every single day.

employees = ["Alice", "Bob", "Carol"]
salary_map = {"Alice": 7500, "Bob": 6000, "Carol": 8200}

for emp in employees:
    pay = salary_map[emp]
    print(f"{emp}: RM{pay}")

# List comprehension — clean, industry-standard style
high_earners = [e for e in employees if salary_map[e] > 7000]
print(high_earners)  # ['Alice', 'Carol']

Beginner Functions & reusable code

Functions are reusable blocks of code. Good developers never copy-paste — they write a function once and call it many times. "Write clean, maintainable code" in job descriptions = this.

def calculate_tax(salary, rate=0.15):
    """Returns tax amount for a given salary."""
    return salary * rate

def net_pay(salary):
    tax = calculate_tax(salary)
    return salary - tax

print(net_pay(7500))   # 6375.0
print(net_pay(10000))  # 8500.0

Intermediate Classes & Object-Oriented Programming

OOP lets you model real things as objects with properties and actions. Almost every developer interview asks about this. Frameworks like Django are built entirely on classes.

class Employee:
    def __init__(self, name, salary):
        self.name = name
        self.salary = salary

    def give_raise(self, percent):
        self.salary *= (1 + percent / 100)
        return self.salary

    def __repr__(self):
        return f"Employee({self.name}, RM{self.salary:.0f})"

alice = Employee("Alice", 7500)
alice.give_raise(10)
print(alice)  # Employee(Alice, RM8250)

Intermediate Error handling & exceptions

Real programs encounter errors — bad input, network failures, missing files. Professional code handles these gracefully instead of crashing. Common interview topic.

import requests

def get_user_data(user_id):
    try:
        response = requests.get(f"/api/users/{user_id}", timeout=5)
        response.raise_for_status()   # raises if 4xx/5xx
        return response.json()
    except requests.exceptions.Timeout:
        print("Server too slow, try again")
    except requests.exceptions.HTTPError as e:
        print(f"HTTP error: {e}")
    except Exception as e:
        print(f"Unexpected error: {e}")
    return None

Intermediate Working with APIs & JSON

Most modern software talks to other systems via APIs. Every job involving data, web, or automation requires this. JSON is the universal language of data exchange.

import requests, json

# Fetch data from a public API
r = requests.get("https://api.github.com/users/octocat")
data = r.json()   # auto-parses JSON into a Python dict

print(data["name"])          # The Octocat
print(data["public_repos"])  # number of repos

# Save to file
with open("user.json", "w") as f:
    json.dump(data, f, indent=2)

Advanced Pandas for data analysis

Pandas is the #1 library in data analyst and data scientist job descriptions. It's Excel but with code — powerful, repeatable, and handles millions of rows instantly.

import pandas as pd

df = pd.read_csv("sales.csv")

# Filter, group, summarise — daily work for any data role
monthly = (df[df["status"] == "completed"]
             .groupby("month")["revenue"]
             .sum()
             .reset_index())

top = monthly.loc[monthly["revenue"].idxmax()]
print(f"Best month: {top['month']} — RM{top['revenue']:,.0f}")

Advanced FastAPI — building REST APIs

FastAPI is one of the most in-demand Python frameworks for backend jobs. It's used to build APIs that mobile apps, frontends, and other services consume.

from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI()

class User(BaseModel):
    name: str
    email: str

fake_db = {}

@app.post("/users/")
def create_user(user: User):
    if user.email in fake_db:
        raise HTTPException(400, "Email already exists")
    fake_db[user.email] = user
    return {"message": "Created", "user": user}

@app.get("/users/{email}")
def get_user(email: str):
    if email not in fake_db:
        raise HTTPException(404, "Not found")
    return fake_db[email]

12-week job-ready roadmap

Study 1–2 hours per day. By week 12 you'll have a portfolio project and be interview-ready.

Weeks 1–2

Python fundamentals

Variables, types, strings, lists, dicts, if/else, for loops, while loops, functions. Build a calculator and a simple quiz game.

Weeks 3–4

OOP, files & modules

Classes, inheritance, reading/writing CSV and JSON files, importing libraries. Build an employee management system.

Weeks 5–6

APIs, error handling & libraries

requests, error handling, virtual environments, pip. Build a weather app and a currency converter using real APIs.

Weeks 7–8

Pick your track

Data: Pandas, NumPy, Matplotlib, Jupyter — analyse a real dataset.
Web: FastAPI or Django — build a REST API with full CRUD operations.

Weeks 9–10

Databases & deployment

SQL basics, connecting Python to PostgreSQL or SQLite, Docker basics, deploying to Railway or Render for free.

Weeks 11–12

Portfolio project + Git + testing

Build one complete project on your chosen track. Write tests with pytest. Push to GitHub. Write a clear README. This is what you show in interviews.

Interview tip: Employers care more about one solid GitHub project than 10 half-finished tutorials. Build something real — even a simple expense tracker or Telegram bot — and be able to explain every line.
